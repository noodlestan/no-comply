## Naming Conventions

| Category    | Convention                                         | Examples                                                              |
| ----------- | -------------------------------------------------- | --------------------------------------------------------------------- |
| Components  | PascalCase                                         | `AppServicesProvider`, `SidebarNav`, `ScreenTemplateBase`             |
| Factories   | `create*`                                          | `createAppNavigation`, `createAppServices`, `createTSXCompilerModule` |
| Hooks       | `use*`                                             | `useAppServices`, `useMeta`, `useUrl`                                 |
| Types       | PascalCase with `API`/`Data`/`Props` suffix        | `AppServicesAPI`, `DocsPageData`, `UIRootProviderProps`               |
| Constants   | UPPER_SNAKE_CASE                                   | `API_BASE_URL`, `FORCED_DELAY`, `$ID_SCREEN_MAIN`                     |
| CSS classes | kebab-case                                         | `has-focus`, `screen-title`                                           |
| Files       | PascalCase for components, camelCase for utilities | `App.tsx`, `createAppNavigation.ts`, `capitalize.ts`                  |

## File Organization Patterns

```
feature/
├── index.ts           # Barrel re-export (auto-generated via // @index directive)
├── FeatureName.tsx    # Main component or function
├── types.ts           # Type definitions
├── constants.ts       # Constants
├── helpers/           # Helper functions
├── parts/             # Sub-components (non-barrel, composed into main)
├── private/           # Internal implementation (not re-exported by index.ts)
├── *.module.scss      # Scoped styles
└── FeatureName.tsx    # Implementation
```

- **Barrel exports** via `// @index` directive comments (tool-generated `index.ts` files)
- **`private/`** subdirectory = internal implementation not exported from the barrel
- **`parts/`** subdirectory = decomposed sub-components of a larger component
- **`helpers/`** subdirectory = utility functions for a feature

## API Design Patterns

### Service Factory Pattern

```typescript
export const createXxx = (): XxxAPI => {
  const api: XxxAPI = { ... };
  return api;
};
```

Used in: `createAppNavigation`, `createAppStatus`, `createTSXCompilerModule`, `createAppServices`

### Context Provider Pattern

```typescript
// Context defined in private/
// Hook with guard in public/
// Provider component in public/
export const XxxProvider: ParentComponent<Props> = props => {
  const service = createXxx();
  return <XxxContext.Provider value={service}>{props.children}</XxxContext.Provider>;
};

export const useXxx = (): XxxAPI => {
  const ctx = useContext(XxxContext);
  if (!ctx) throw new Error('useXxx() must be wrapped in <XxxProvider/>');
  return ctx;
};
```

Used in: `AppServicesProvider`, `MetaProvider`, `CodeBlockProvider`, `UIRootProvider`

### Typed API Objects over Classes

- All services return plain object type aliases (`AppNavigationAPI`, `AppStatusAPI`, `AppServicesAPI`)
- No classes — factory functions return typed objects
- SolidJS `Accessor<T>` for reactive values

### Module `private/` Directory Pattern

```
providers/AppServices/
├── index.ts              # Exports: AppServicesProvider, types, useAppServices
├── AppServicesProvider.tsx
├── types.ts
├── useAppServices.tsx
└── private/              # NOT re-exported by index.ts
    ├── index.ts
    ├── AppServicesContext.tsx
    └── createAppServices.ts
```

## Typing Conventions

- **Typed API objects** — All service APIs are plain object types (`AppNavigationAPI`, `AppStatusAPI`)
- **Generics** — Used sparingly; prefer concrete types
- **No classes** — Pure factory functions returning typed objects
- **SolidJS Accessor** — `Accessor<T>` for all reactive values
- **Component Props** — PascalCase with `Props` suffix: `UIRootProviderProps`
- **Context types** — Defined in `private/` and consumed via guarded hooks

## Styling Conventions

- **CSS Modules** (`*.module.scss`) for component-scoped styles
- **SCSS** with global variables/mixins injected via Vite `additionalData`
- Utility: `createClassList` from `@no-comply/solid-primitives` for computed class lists
- `@media (prefers-color-scheme: light)` in `index.html` for color scheme base styles
- Base styles: `layers.css` + `reset.css` in `src/styles/`
