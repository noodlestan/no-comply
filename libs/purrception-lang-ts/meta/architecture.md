## Design Principles

1. **Discriminated union as core abstraction** — All type representations (`TypeExpressionNode`, `Declaration`, `DeclarationKind`) use a `kind` string discriminator, enabling exhaustiveness checking and switch-based dispatch.
2. **Separation of data from behavior** — Types defined in `types.ts` files; operations in separate function modules.
3. **Immutable transformations** — All resolvers produce new objects via spread; no in-place mutation.
4. **Explicit context passing** — No global state; `ResolveTypeContext` bundles all state and callbacks for recursive resolution.
5. **Graceful degradation** — Resolution failures produce `console.warn` and return the original node rather than crashing. Circular references throw.

## Dependency Flow

- **Strictly acyclic downward**: `constants` → `jsdoc` → `node` → `declaration` → `resolve`
- No module depends on a module at the same or higher layer
- No circular dependencies between modules
- `resolve` imports from all lower layers and from `@purrception/primitives`

## Extensibility

- **New type expression kinds**: Add to `TypeExpressionKind` union → define type → add case in `resolveExpression` switch → write resolver
- **New declaration kinds**: Add to `DeclarationKind` union → define type → add type guard
- **New operators**: Add case in `resolveOperator` → add resolver in `operators/`
- Design is **open for extension within the module** but **closed for external extension** (no plugin system)

## Key Trade-offs

| Decision                      | Rationale                                                                 |
| ----------------------------- | ------------------------------------------------------------------------- |
| 18 type expression kinds      | Covers vast majority of TS types without modeling every possible AST node |
| Switch-based dispatcher       | Simple, explicit, easy to reason about — harder to extend externally      |
| Immutable spreads + recursion | Safer and predictable; potential perf cost for deeply nested types        |
| `console.warn` + return node  | Graceful degradation for IDE-like contexts                                |
