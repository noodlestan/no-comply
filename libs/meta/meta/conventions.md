## File Organization Patterns

- **Entity-per-directory**: Each entity type in `src/entities/<EntityName>/` with consistent internal structure:
  ```
  <EntityName>/
    index.ts        — re-exports helpers and types
    types.ts        — entity-specific type definitions
    helpers/
      index.ts      — re-exports all helpers
      isNoComply*.ts — type guard
      ...            — entity-specific helpers
  ```
- **Barrel exports**: Every directory has `index.ts` re-exporting submodules. Files contain `// @index(...)` comments suggesting barrel generation via `@index` tool.
- **Private module pattern**: Internal implementation goes in `src/private/` and is not re-exported through the public entry point.
- **Separation of concerns**: Types (`types.ts`) separated from implementation logic; helpers separated from entity modules.

## API Design Patterns

- **Service factory pattern**: `createNoComplyMetaService` is a factory function returning a query object — no classes.
- **Maybe/Option pattern**: Every lookup has both a safe variant (`getEntityMaybe` returns `undefined`) and a throwing variant (`getEntity` throws on miss).
- **Type predicate guards**: All entity guards are proper TypeScript type predicates (`dec is ComponentEntityData`).
- **Generic filter pattern**: `getEntityFunctions` / `getEntityTypes` accept a `Filter` union — array of names (whitelist) or predicate function.
- **Defensive copying**: `getEntities()` returns `[...entities]` spread copy to prevent mutation.
- **Fail-fast on duplicates**: `indexEntities` throws on duplicate entity keys.
- **Consistent error messages**: Format follows `"Could not resolve ..."`, `"Unknown entity ..."`, `"Invalid entity"`, `"Duplicate entity ..."`.

## Typing Conventions

- **Composition over deep inheritance**: Entity types are intersection types, not class hierarchies.
- **Strict type discrimination**: Literal `type` field enables discriminated union checks (`dec.type === 'component'`).
- **Generic service API**: `NoComplyMetaAPI` uses `<T extends NoComplyEntityData>` for type-safe retrieval.
- **External type reuse**: Core primitives from `@purrception/primitives` are extended, not redefined.
- **No runtime enums**: Type discriminators are string literal unions, not enum values.
