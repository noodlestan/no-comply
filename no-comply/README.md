# @no-comply

> Context-aware UI system built with SolidJS and modern CSS.

No Comply is a layered library ecosystem spanning foundational types, accessibility primitives, contexts and servicers, composable controllers, a themeable component library, and developer tooling.

## Packages

### Libraries

- [@no-comply/solid-primitives](./libs/solid-primitives/README.md) — Types and prop helpers.
- [@no-comply/solid-accessibility](./libs/solid-accessibility/README.md) — Types, and role controllers.
- [@no-comply/solid-contexts](./libs/solid-contexts/README.md) — Contexts, services and providers for building apps.
- [@no-comply/solid-composables](./libs/solid-composables/README.md) — Controllers, mixins and unstyled components.
- [@no-comply/standard-ui](./libs/standard-ui/README.md) — Themeable component library.
- [@no-comply/solid-dev-tools](./libs/solid-dev-tools/README.md) — WIP debug/instrumentation tools.

### Apps

- [@no-comply/standard-ui-demo](./apps/standard-ui-demo/README.md) — Demo application and documentation.

### Supporting libraries

- [@no-comply/meta](./libs/meta/README.md) — Purrception types, services, helpers (private, not published).

### CLI

- [@no-comply/meta-extract](./cli/meta-extract/README.md) — (CLI) orchestrate extraction.
